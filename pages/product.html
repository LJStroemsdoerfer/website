Effortless Machine Learning Deployments

Deploying Machine Learning models can be super tedious. Depending on where you want to deploy, your model will need to cross two, three, sometimes four or five differnet ecosystems in order to get to it's final productive stage.

Most often, the machine learning model itself, does not make up for the entire final product, but rather is a part of something bigger. Whether you have some additional database, a frontend through which your end user interacts with your model, or whatever additional service.

The state-of-the-art orchestration and hosting solution for such interdisciplinary products is Kubernetes. It is a super powerful and quite efficient way to manage products, that consist of a multitude of different pieces.

While most often being the platform of choice, Kubernetes is quite complex and so, Data Scientists are forced to take on a new set of skills and most importantly tasks. This will inevitably slow down the pace and reduce productivity. With our till engine, Data Scientists can easily turn Machine Learning into Kubernetes deployable software pieces. And the best part, without spending hours and hours making sense of Kubernetes documentation.

The till engine automates and abstracts the deployment steps, that would involve any ecosystem other than your preferred IDE. At the core of our engine, we make use of REST to build APIs and Docker for containerization.

Python API

Our Python API enables a stunningly fast construction of Docker containers. With a full integration into the Flask API framework, Pythonists can construct, build, run and deploy containers without leaving Python.

Data Scientists can build their API in Flask and till will take care of catching the necessary libraries and file resources.

In addition to the build() and run() methods, you have a deploy() method, which allows you to ship your API to any remote registry.

till is available on PyPi. So you can download the library as every other Python library, using: pip install till

R API

Our R API allows you to construct Docker containers directly from RStudio. There is no need to take any detours or extra effort. Our package integrates perfectly withe the most promiment R API framework plumbeR. R Data Scientists can simply build their plumbeR API as they are used.

till takes care of all the heavy lifting. It checks all necessary package dependencies, as well as file resources that are needed to run the API.

We comply with the standard R interface, instead of running plumbeR run commands, you can use the till functions to get your API containerized.

Similar to the Python API you can not only build and run containers, but also deploy them to any remote location and all of this can be done, from your favorite R IDE.

To install till you can simply use your standard CRAN mirror and run the typical installation command: install.packages('till').
